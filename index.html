detection rules

Rules that fire on purpose

Custom Wazuh detection rules, each mapped to a MITRE ATT&CK technique.

Windows Command Detection Set
WAZUH · 5 RULES

Five custom rules detecting common Windows commands used in discovery and execution stages, each restricted to firing only on Sysmon process-creation events (parent SID 61603).

<!-- rules/local_rules.xml --> <group name="custom_windows,"> <!-- PowerShell execution --> <rule id="100011" level="10"> <if_sid>61603</if_sid> <match>powershell.exe</match> <description>PowerShell execution detected</description> <mitre><id>T1059.001</id></mitre> </rule> <!-- cmd.exe, whoami, systeminfo, ipconfig follow the same pattern --> <rule id="100010" level="8"> <if_sid>61603</if_sid> <match>cmd.exe</match> <description>CMD execution detected</description> <mitre><id>T1059.003</id></mitre> </rule> </group>
